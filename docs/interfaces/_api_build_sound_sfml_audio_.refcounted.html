ref

  • ref(): number
    • Defined in API/build/Sound/sfml_audio.ts:10

    Increments the reference count.

    After this call the reference will be a strong reference because its refcount is >0, and the referenced object is effectively "pinned". Calling this when the refcount is 0 and the object is unavailable results in an error.

    Returns number

    Current reference count

  • release(): void

    Deletes a reference.

unref

  • unref(): number
    • Defined in API/build/Sound/sfml_audio.ts:20

    Decrements the reference count.

    If the result is 0 the reference is now weak and the object may be GC'd at any time if there are no other references. Calling this when the refcount is already 0 results in an error.

    Returns number
